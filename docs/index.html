Welcome to msTools’s documentation!¶

Structure¶

The msTools package is split into multiple sub-packages containing either core or specialised functionality. Core functionality is split between the msTools.core and msTools.coreUI sub-packages. Both provide low-level abstractions for various APIs. Specialised packages such as msTools.tools and msTools.metadata are dependant upon core functionality but have a higher-level objective.

Guidelines¶

  1. Operations will preference throwing an exception over returning a null OpenMaya object.
  2. Operations will assume the internal Maya object of any OpenMaya input is valid.
  3. Operations will raise a msTools.core.maya.exceptions.MayaTypeError for any exceptional behaviour relating to the internal type of Maya objects. This attempts to homogenise the behaviour of the OpenMaya API which raises TypeError or RuntimeError for different object types.

Syntax¶

This documentation makes use of a legacy approach to specifying types since static type checking was implemented in Python 3.5 (see typing module), whilst msTools is confined to Python 2.7.
Because there is no standard type syntax for this legacy approach, use of the PyCharm type syntax is adopted with the following amendments:
Syntax Description
any Argument of any type.
Foo, optional Argument of type Foo which has a default value assigned.
callable[[Foo, Bar], Bar] A callable with two arguments [Foo, Bar] and return type Bar.
callable[..., Bar] A callable with any call signature and return type Bar.
iterable[Foo] Any iterable type containing elements of type Foo.
(Foo, Bar) A tuple of two elements whose types are Foo and Bar respectively.
(Foo, ...) A variable length tuple containing elements of type Foo.
T <= Foo Argument of type T with upper bound Foo, meaning T is a (non-strict) subclass of Foo.

Note

If an optional argument of type Foo accepts a default value of None this will be mentioned in the description as opposed to displaying a union of types Foo | None.
